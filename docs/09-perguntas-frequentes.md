# 09 — Perguntas Frequentes (FAQ) ❓

Abaixo, as respostas para as dúvidas mais comuns de quem está começando a usar a nossa plataforma:

---

### 1. Posso usar este projeto sem saber programar? 🖥️
**Sim!** Este guia foi escrito para que você não precise digitar nenhuma linha de código. Você só precisará copiar, colar e ajustar alguns nomes e senhas no servidor.

### 2. Preciso de um servidor dedicado ou VPS serve? 🏢
**Uma VPS (Servidor Virtual) é o ideal.** Ela é muito mais barata que um servidor físico e aguenta centenas de conexões simultâneas com tranquilidade se você seguir os passos de configuração mínima recomendada.

### 3. Quantos computadores (clientes) posso gerenciar? 📊
Com a configuração recomendada de **vCPU/RAM**, você consegue gerenciar de **500 a 1.000 computadores** sem lentidão. Se sua empresa crescer, basta aumentar a potência do seu servidor.

### 4. O RustDesk funciona em Mac e Linux além de Windows? 🍎🐧
**Sim!** Ele funciona em quase tudo: Windows, macOS, Linux, Android e até no iOS (apenas visualização). Este guia foca no cliente Windows, que é o mais comum, mas a lógica é a mesma para os outros.

### 5. O que acontece se o servidor cair? ⚠️
Se o seu servidor desligar por falta de pagamento ou erro técnico, os técnicos **não conseguirão acessar** os computadores dos clientes até que o servidor volte à vida. Por isso é importante usar empresas confiáveis.

### 6. Como migro do TeamViewer para esta plataforma? 🚚
O processo é simples:
1. Instale o nosso cliente no computador do cliente final.
2. Certifique-se de que ele aparece no seu painel.
3. Desinstale o TeamViewer. Pronto!

### 7. Posso ver a tela do cliente sem ele saber? 🕵️‍♂️
**Por definição, o RustDesk avisa ao cliente que você está conectado.** No entanto, para fins de administração interna, é possível configurar o acesso desassistido (com senha fixa), mas sempre haverá um indicativo visual de que a tela está sendo compartilhada por questões de ética e segurança.

### 8. Quanto de internet o servidor consome? 🌐
O consumo é baixo, pois a conexão tenta sempre ser direta entre o técnico e o cliente. O servidor só "trabalha mais" quando os dois computadores não conseguem se enxergar (usando o modo **Relay**). Uma VPS comum tem banda de sobra para isso.

### 9. Onde posso pedir ajuda se tiver problemas? 🆘
Você pode consultar a nossa própria documentação, a comunidade oficial do RustDesk no GitHub, ou se você for um cliente corporativo, entrar em contato com o suporte da sua empresa.
