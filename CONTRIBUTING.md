# 🤝 Guia de Contribuição

Ficamos muito felizes que você tenha interesse em contribuir com a nossa Plataforma de Suporte Remoto! Este projeto foi construído para ser modular, seguro e fácil de evoluir. Mesmo que você seja iniciante, há várias formas de ajudar.

---

## 🌟 Como Começar?

Se você é novo no projeto ou no mundo Open Source, aqui estão os primeiros passos:

1.  **Entenda o Projeto**: Leia nossa [Visão do Produto](specs/vision.md) para entender que problema estamos resolvendo.
2.  **Explore a Documentação**: Temos guias específicos para cada área no diretório `docs/`.
3.  **Ambiente Local**: Siga o passo-a-passo no [Guia de Onboarding](docs/onboarding.md) para rodar o projeto na sua máquina.

---

## 🛠️ O que posso fazer?

Não precisa saber programar tudo para contribuir! Escolha o que mais combina com você:

### 📝 Documentação (Iniciantes)
- Corrigir erros de digitação ou gramática.
- Melhorar explicações que ficaram confusas.
- Traduzir partes do projeto.

### 🐛 Reportar Bugs (Todos)
- Encontrou algo que não funciona? Abra uma **Issue** explicando o que aconteceu e como podemos reproduzir o erro.

### 💻 Sugerir Funcionalidades
- Tem uma ideia incrível? Crie uma Issue com a tag `enhancement` descrevendo sua sugestão.

### 👩‍💻 Resolver Issues
- Procure por Issues com a tag `good first issue`. Elas são perfeitas para quem está começando!

---

## 🚀 Enviando sua Contribuição (Pull Request)

Quando estiver pronto para enviar seu código:

1.  **Crie um Fork** do repositório.
2.  **Crie uma Branch** para sua mudança: `git checkout -b feature/minha-melhoria`.
3.  **Faça seus Commits**: Use mensagens claras em português.
4.  **Envie o Push**: `git push origin feature/minha-melhoria`.
5.  **Abra um Pull Request**: Explique o que você mudou e por quê.
